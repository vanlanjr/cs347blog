Week 4 Blog

Posted on October 1, 2020

This week in web development, I learned about how to have a responsive web design using media queries. This taught me to create website that has a good viewport for both desktop and mobile screens. Once I learned how to do this, I implemented it in my project 1 assignment webpage. For the lab on this day, my group and I experimented with using media queries to keep the display looking elegant while the browser is resized. This lab was pretty difficult and my group wasn't able to finish it in class.

The next lecture was about using the grid layout for manipulating forms. These forms can be designed on the webpage or they can be taken in as input from the website's user. Additionally, I learned how to create a list of automatic suggestions for text inputs. This is more advanced and looks nicer than just providing a plain input text box. The last topic that was covered was creating labels that stay in the field of input. Using various CSS techniques, these labels can be subject to outline on click and have a defined background from the body.

Outside of class, I did some research on the Locomotive Scrolling bar provided on GitHub. In order to do this, I watched a video called Locomotive Scroll Is Awesome - Here's How to Use it. To apply locomotive scroll to my webpage, I had to download the package using npm from Node.js. My implementation includes javascript as well as new html tags.

Here is the code the scrolling bar:

JavaScript:
import LocomotiveScroll from 'locomotive-scroll';

const scroll = new LocomotiveScroll({
el: document.querySelector('[data-scroll-container]'),
smooth: true
});

CSS:
* {
box-sizing: border-box;
}
main {
height: 100vh;
margin: 0;
}
section {
height: 100vh;
}
