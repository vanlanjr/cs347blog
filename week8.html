Outside of class, I decided to do more research on JSX and how it is different from JavaScript. JSX is used in React apps as a way to create React elements. The article I read is called JSX In Depth. I learned a hand full of new information and best practices when it comes to using JSX in React apps to crete elements.

Here are a few of those things:

  • The first type of the JSX tag specifies the type of the React element. 
                    React.createElement(
                  MyButton,
                  {color: 'blue', shadowSize: 2},
                  'Click Me'
                )
    is a MyButton React component because of the capitalized first letter, and 
                    React.createElement(
                  'div',
                  {className: 'sidebar'}
                )
    is a div.
  • Components created by the user should be capitalized and they should also be capitalized when they are referenced in code.
  • JavaScript expressions can be passed as props if they are enclosed in {}.
  • You can use ...props as a "spread" operator in a prop that will pass the whole props object. The function: 
                    function App() {
                  const props = {firstName: 'Ben', lastName: 'Hector'};
                  return ;
                }
    will return the whole props object.
  • You can access a component's children by calling props.children. The children can be a string literal, other components, arrays, or JavaScript expressions.
  • Booleans, null, and undefined are ignored in React. If you want to use these values to appear in output, then you need to convert them to a string first. This would look like: 
                    div>
                  My JavaScript variable is {String(myVariable)}.
                /div>
